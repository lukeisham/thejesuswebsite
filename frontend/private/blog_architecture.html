Step 1 & 2 — CRUD Editor + Modal Workflow

 dashboard.html — Blog CRUD panel (id: crud-blogs)
 ├─ #blog-list (UL)
 │   Posts load automatically on panel init via loadPosts()
 │   GET /api/blog/posts → renderList(posts)
 │   Each row: title | "Published" badge OR "Draft" badge
 │
 ├─ #search-blogs-input
 │   keyup → filters #blog-list rows by title in real-time
 │
 ├─ #create-blog-post button
 │   └─ clearModal() → opens #blog-editor-modal  (new post)
 │
 └─ click any post row
     └─ populateModal(post) → opens #blog-editor-modal  (edit existing)

         ┌────────────────────────────────────────────────────┐
         │  #blog-editor-modal  (popup overlay)               │
         │                                                    │
         │  #blog-edit-title    textarea — post title         │
         │  #blog-edit-body     textarea — Markdown body      │
         │                                                    │
         │  #blog-save-draft  ──┐                             │
         │  #blog-publish     ──┤──→ saveOrPublish(bool)      │
         │  #blog-close-btn   ──┘     │                       │
         └──────────────────────────────────────────────────┘
                                      │
                                      ▼
                         POST /api/blog/posts
                         { title, body, published: bool }
                                      │
                                      ▼
                         api_blog.rs: handle_create_post()
                         ├─ BlogCreateRequest → DraftBlogPost
                         ├─ Gatekeeper::validate_draft()
                         │   ├─ title must not be empty
                         │   └─ body must be ≥ 10 characters
                         ├─ new BlogPostId (UUID) + Metadata (ULID)
                         ├─ ⚠ security: is_published forced to false
                         │   (regardless of published field sent)
                         └─ chroma.rs: store_blog_post()
                             → embeds title + body text
                             → upserts into "blog_posts" collection

Step 4 — Most Recent Post (news_and_blog.html)

 news_and_blog.html (page load)
 ├─ js/display_top_blog_post.js → initTopBlogPost()
 │       │
 │       ▼
 │  GET /api/blog/latest   ← INTENDED — backend not yet implemented
 │       │                    (handler to be added in api_blog.rs;
 │       │                     queries ChromaDB for most recent post)
 │       ▼
 │  Expected response:
 │  { category, title, excerpt, body, author, date, url }
 │       │
 │       ▼
 │  Populates #top-blog-post section:
 │  ├─ .essay-kicker  ← category  (e.g. "Featured")

 │  ├─ .essay-meta    ← "By {author} · {date}"

 └─ js/display_top_four_news_items.js  (co-loaded — handles news section)

File & Function Reference

Backend — Core Types
FileExport / TypeDescription
app_core/src/types/blog_and_news/blog.rs BlogPost struct 7 fields: id, title, content (→ "body" in JSON), picture_url (Option), labels, metadata, is_published (→ "published")
BlogPostId(uuid::Uuid)Newtype wrapper around UUID — used as unique post identifier
DraftBlogPost structIntermediate: title, content, picture_url, labels — no id or status yet
Gatekeeper::validate_draft(draft)title non-empty · body ≥ 10 chars · returns BlogPostId or BlogError
BlogErrorNotFound | ValidationError(String) | SecurityViolation(String) | LockError
app_core/src/types/dtos.rs BlogCreateRequest API input DTO: { title, body, published, picture_url?, labels }
From<BlogCreateRequest> for DraftBlogPostMaps body → content · drops published flag (handled separately)
Backend — API Handlers
HandlerRouteMethodStatusDescription
handle_get_posts/api/blog/postsGET✅Queries ChromaDB with empty string → returns Vec<BlogPost> JSON
handle_create_post/api/blog/postsPOST✅BlogCreateRequest → DraftBlogPost → BlogPost · stores in ChromaDB · is_published forced false
handle_get_latest/api/blog/latestGET⚠️ intendedReturns most recent post for featured slot on news_and_blog.html · not yet built
handle_blog_feed_content/api/v1/blog_feed_contentGET⚠️ intendedReturns HTML fragment of post cards for blog_feed.html · not yet built

Implemented handlers in app/app_ui/src/api_blog.rs. Routes registered via nest("/api/blog", blog_routes()) in router.rs.

Backend — Storage (ChromaDB only)
FileMethodDescription
app_storage/src/chroma.rs store_blog_post(post) Embeds title + body text → upserts into "blog_posts" ChromaDB collection by UUID
query_blog_posts(query_text)Semantic similarity search on "blog_posts" collection → returns Vec<String> (JSON docs)

⚠️ Blog posts are stored in ChromaDB only — sqlite.rs has no blog methods. The previous architecture diagram incorrectly showed sqlite.rs as the storage layer.

Glossary of Terms

TermDefinition
BlogPostRust struct with 7 fields: id (UUID), title, content (serialised as "body"), picture_url (optional), labels, metadata, is_published (serialised as "published")
DraftBlogPostIntermediate struct used before ID assignment: title, content, picture_url, labels — no id or publication status
BlogCreateRequestAPI input DTO sent from the dashboard: { title, body, published, picture_url?, labels }
MarkdownLightweight text formatting syntax written in #blog-edit-body (headers, bold, italic, links, lists)
is_publishedBoolean flag on BlogPost; always forced to false at creation by the backend regardless of what the client sends
ChromaDBVector database storing all blog posts in the "blog_posts" collection via semantic embedding of title + body
blog_posts collectionChromaDB collection holding embedded blog post documents; queried by handle_get_posts() and (intended) handle_blog_feed_content()
blog_crud.jsFully implemented dashboard CRUD script: loads post list on init, real-time search filter, modal editor, save/publish API calls
blog-editor-modalPopup overlay (#blog-editor-modal) in dashboard.html containing the title textarea, Markdown body textarea, and action buttons
blog_feed.htmlPublic blog grid page — injects all published post cards into #hero-placeholder via blog_feed_hero.js
news_and_blog.htmlPublic page showing top 4 news items alongside the most recent blog post in a featured #top-blog-post slot
hero-placeholderContainer <section> in blog_feed.html that receives dynamically injected blog post card HTML from blog_feed_hero.js
top-blog-postFeatured section in news_and_blog.html populated by display_top_blog_post.js with the most recent post's title, excerpt, author, date, and link
