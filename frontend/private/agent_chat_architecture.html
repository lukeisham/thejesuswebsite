Agent & User Chat Architecture

Reference for chat interaction modes, widget commands, and the heavy/light action system.

Data Flow

Interaction Mode Flow

User message → detectInteractionMode() [frontend JS]
  → "Execution"    — brief commands, no preamble
  → "Collaborative" — iterative, suggest 3 alternatives (default)
  → "Review"       — methodical, flag anomalies
  → "Monitor"      — short, alert-aware

→ forwarded as interaction_mode field in POST /api/v1/agent/chat
→ InteractionMode::from_str() [agent.rs]
→ vibe suffix appended to AgentResponse.data

Heavy vs Light Actions

HEAVY ACTIONS (require verification)     LIGHT ACTIONS (execute immediately)
─────────────────────────────────────    ────────────────────────────────────
"run wiki"   → run_wiki_sync             "run news"    → run_news_crawler
"run challenge" → run_challenge_sort     "run spelling"→ run_spelling_check
"run deadlink"  → run_deadlinks          "run scrape"  → run_page_scraper
"run embed"  → run_vector_embed

Returns: verification_required: true     Returns: action string, executes now
