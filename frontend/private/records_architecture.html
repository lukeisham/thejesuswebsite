Record Toggle — Bible Verse Links

 The Record Toggle turns every Bible verse on any page into a clickable
 internal link that navigates the user to records.html for that record.

 ┌──────────────────────────────────────────────────────────────────┐
 │  ANY PAGE (evidence, timeline, maps, essays, list pages, etc.)  │
 │                                                                 │
 │  Bible verses rendered with class="record-link" (hidden)        │
 │  e.g. <a class="record-link hidden"                            │
 │         href="/records.html?verse=Mk+4:35-41">                 │
 │         Mk 4:35–41                                              │
 │       </a>                                                      │
 └──────────────────────────┬───────────────────────────────────────┘
                            │
                 User clicks footer button
                 #btn-toggle-links
                            │
                            ▼
               footer_actions.js
               toggleRecordLinks()
                            │
                            ▼
               document.querySelectorAll('.record-link')
               .forEach(link => link.classList.toggle('hidden'))
                            │
                            ▼
 ┌──────────────────────────────────────────────────────────────────┐
 │  TOGGLE ON:  Bible verses become visible clickable links        │
 │              "Mk 4:35–41" → click → /records.html?verse=…       │
 │              records.html reads ?verse param → filters to        │
 │              the matching record and displays it                 │
 │                                                                 │
 │  TOGGLE OFF: Links hidden again, verses show as plain text      │
 └──────────────────────────────────────────────────────────────────┘

 Key files:
 ├─ _footer.html          → contains #btn-toggle-links button
 ├─ js/footer_actions.js  → toggleRecordLinks() function
 ├─ style.css             → .record-link + .hidden classes
 └─ records.html          → reads ?verse query param on load

Glossary of Terms

TermDefinition
ULIDUniversally Unique Lexicographically Sortable Identifier — the primary key for every record
RecordA 14-field structured entry representing a biblical event, person, place, or theme
Primary VerseThe main Bible reference associated with a record (e.g., "Mk 4:35–41")
Secondary VerseAn optional additional Bible reference for supplementary context
Internal LinkAn <a> tag linking to records.html with a query parameter to identify the target record
Record ToggleFooter button (#btn-toggle-links) that makes Bible verses across the site clickable links to their associated record
DB PopulatorDashboard widget (wgt_db_populator.js) that seeds ~300 records by parsing the 6 list pages
ClassificationRecord category enum: Event | Location | Person | Theme (PascalCase serde)
ContentEntryRecord content type: Miracle | Parable | Saying | Sermon | Other (lowercase serde)
TimelineEraOne of 8 chronological periods: pre-incarnation, birth-early-life, baptism-preparation, galilean-ministry, judean-ministry, passion-crucifixion, resurrection-ascension, theme
MapTypeGeographic region enum: Galilee | Jerusalem | Judea | Levant | Rome | Overview
ChromaDBVector database used for semantic search of records (stores embedded JSON in "records" collection)
CRUD EditorDashboard form (edit_records.js) for Creating, Reading, Updating, and Deleting records
RecordGatekeeperRust validator ensuring record data integrity before storage (name ≤ 80 chars, PNG format, non-empty description)
