Agent Backend Integration Architecture

Reference for how the agent brain, intent classifier, and work queue are wired into the Rust server.

Data Flow

Agent Request Flow

POST /api/v1/agent/chat
  → handle_agent_chat() [agent_api.rs]
  → AppState.brain (Option<Arc<CandleEngine>>)
  → if None: return "AI Agent is currently offline"
  → if Some: Agent::new(brain) → orchestrate_with_context()
    → sanitize() [Gatekeeper]
    → determine_intent() [keyword classifier]
    → dispatch to engine (Wiki / Challenge / Essay / Record / Response / WidgetCommand)
    → AgentResponse { data, confidence, metadata }

Work Queue Flow

WorkItem (HumanRequest | AgentReferral)
  → WorkQueue (VecDeque, FIFO, capacity-gated)
  → GET /api/v1/agent/queue           [retrieve list]
  → POST /api/v1/agent/queue/run-next [consume head]
