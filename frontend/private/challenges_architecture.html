Academic vs Popular — Ranking Comparison

Both pipelines share the same code path. The only two differences are the search context (what sources are queried) and the ranking algorithm (what the score measures). 

  ┌─────────────────────────────────┬──────────────────────────────────┐
  │          ACADEMIC               │           POPULAR                │
  ├─────────────────────────────────┼──────────────────────────────────┤
  │                                 │                                  │
  │  Search Context                 │  Search Context                  │
  │  ───────────────                │  ──────────────                  │
  │  AcademicChallengeTool          │  PopularChallengeTool            │
  │  query: "academic"              │  query: "trending"               │
  │  sources: academic literature,  │  sources: trending/popular       │
  │  peer-reviewed journals,        │  media, social platforms,        │
  │  scholarly publications         │  mainstream news                 │
  │                                 │                                  │
  │  Ranking Algorithm              │  Ranking Algorithm               │
  │  ─────────────────              │  ─────────────────               │
  │  AcademicRanker                 │  PopularRanker                   │
  │  Academic(u8) score 1-100       │  Popular(u8) score 1-100         │
  │  measures: sustained presence   │  measures: current popularity    │
  │  over time in academic          │  on mainstream and social        │
  │  literature                     │  media platforms                 │
  │                                 │                                  │
  │  Storage                        │  Storage                         │
  │  ───────                        │  ───────                         │
  │  Raw: challenges_raw_academic   │  Raw: challenges_raw_popular     │
  │  Promoted: challenges_academic  │  Promoted: challenges_popular    │
  │                                 │                                  │
  │  Rust Types                     │  Rust Types                      │
  │  ──────────                     │  ──────────                      │
  │  RawAcademicChallenge           │  RawPopularChallenge             │
  │  AcademicChallenge              │  PopularChallenge                │
  │  Academic(u8)                   │  Popular(u8)                     │
  │                                 │                                  │
  │  Pipeline Traits                │  Pipeline Traits                 │
  │  ───────────────                │  ───────────────                 │
  │  ChallengeGenerator impl        │  ChallengeGenerator impl         │
  │  ChallengeProcessor impl        │  ChallengeProcessor impl         │
  │                                 │                                  │
  │  Source File                    │  Source File                     │
  │  ───────────                    │  ───────────                     │
  │  app_brain/src/challenge.rs     │  app_brain/src/challenge.rs      │
  │                                 │                                  │
  │  Balanced Check                 │  Balanced Check                  │
  │  ──────────────                 │  ──────────────                  │
  │  ChallengeRank::is_balanced()   │  ChallengeRank::is_balanced()    │
  │  popular - academic ≤ 10        │  popular - academic ≤ 10         │

Glossary of Terms

TermDefinition
RawChallengeAn unvalidated challenge entry-point combining both RawPopularChallenge and RawAcademicChallenge before promotion through the ranking pipeline.
PopularChallengeA validated challenge sourced from trending/popular media. Has url, name, metadata, and a Popular(u8) ranking score 1-100.
AcademicChallengeA validated challenge sourced from academic literature. Has url, name, metadata, and an Academic(u8) ranking score 1-100.
ChallengeRankAggregate struct containing both Popular and Academic scores. is_balanced() checks if the two scores differ by ≤ 10.
ChallengeLinkA reference struct with url and name fields. Used by Response to link back to its parent challenge.
ChallengeListContainer for all challenge states (raw_popular, popular, raw_academic, academic). Provides promote_all() and security_scan().
ResponseA written reply to a specific challenge. Has internal_url, title, text, linked_challenge (ChallengeLink), parent_id, and publication status.
internal_urlAuto-generated URL for a response derived from its title slug. Must use the vibe.internal domain (e.g., vibe.internal/free-will-defense).
PublicationStatusEnum controlling visibility: Unpublished (default on creation) or Published (after Response::publish() validation).
ChallengeGeneratorAsync trait in app_brain with generate(count) method. Implemented by PopularChallengeTool and AcademicChallengeTool.
ChallengeProcessorAsync trait in app_brain with process_pipeline(raw) method. Implemented by PopularRanker and AcademicRanker.
PopularRankerImplements ChallengeProcessor for popular challenges. Ranks by current popularity on mainstream/social media, then enriches with metadata.
AcademicRankerImplements ChallengeProcessor for academic challenges. Ranks by sustained presence in academic literature, then enriches with metadata.
wgt_challenge_rankerFrontend widget (wgt_challenge_ranker.js) that provides a dashboard control for triggering challenge sort via POST /api/v1/tools/challenge/sort. Auto-polls every 60 seconds.
response_countNumber of published responses linked to a challenge. Displayed alongside each challenge title in the CRUD list.
