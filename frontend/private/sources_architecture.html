Sources Architecture

Three-stage pipeline: validate a source, publish it to list_sources.html, then scan site content to confirm it is cited.

Data Flow

Stage 1 — Validate Source

User enters source in Sources Widget (wgt_sources.js)
  │
  ▼
┌─────────────────────────────────────────────────────────────────────┐
│  validateSource(input)  ← JS script: source_validator.js           │
│                                                                     │
│  Parse the following structure:                                     │
│                                                                     │
│  title ─┬─ URL            e.g. https://jstor.org/article/123       │
│         ├─ <i>Book Name</i>   e.g. <i>The Historical Jesus</i>    │
│         └─ 'Article Name' e.g. 'Criteria of Authenticity'          │
│                                                                     │
│  IF title is an 'Article' AND a journal follows:                    │
│     → journal in <i>italics</i>                                    │
│                                                                     │
│  "by" ── author ─┬─ First Last   e.g. John Meier                  │
│                   └─ ORCID link   e.g. 0000-0002-1825-0097         │
│                      → author name becomes <a href="orcid.org/…">  │
│                                                                     │
│  (year)  e.g. (2009)                                                │
│                                                                     │
│  THEN append identity <span> link:                                  │
│     IF article → DOI   e.g. <span>doi:10.1093/…</span>            │
│     IF book    → ISBN  e.g. <span>ISBN 978-0-300-…</span>         │
│                                                                     │
│  Output: validated Source struct → POST /api/v1/admin/sources       │
└─────────────────────────────────────────────────────────────────────┘
│
▼
┌─────────────────────────────────────────────────────────────────────┐
│  Rust Gatekeeper: SourceGatekeeper::new()                           │
│  ├─ title.text: non-empty, ≤ 140 chars                             │
│  ├─ author: Name(string) or Orcid(string), non-empty               │
│  ├─ identity: NamedUrl | Isbn | AcademicArticleId (DOI)            │
│  └─ → Ok(Source) persisted to SQLite via create_source()           │
└─────────────────────────────────────────────────────────────────────┘

Stage 2 — Publish to list_sources.html

GET /api/v1/sources → returns all Source objects from SQLite
  │
  ▼
┌─────────────────────────────────────────────────────────────────────┐
│  list_sources_hero.js  ← JS script: source_publisher.js            │
│                                                                     │
│  1. Fetch all sources from /api/v1/sources                         │
│  2. Group by category (if grouped) or render flat list             │
│  3. For each source, call formatSource(source):                    │
│     ├─ Author::Name("John Meier")  → plain text                   │
│     ├─ Author::Orcid("0000-…")     → <a href="orcid.org/…">      │
│     ├─ SourceTitle.text             → rendered with type styling   │
│     ├─ SourceIdentity::AcademicArticleId → <span>DOI link</span> │
│     ├─ SourceIdentity::Isbn             → <span>ISBN</span>      │
│     └─ SourceIdentity::NamedUrl         → <span>URL link</span>  │
│  4. Inject rendered HTML into #hero-placeholder on list_sources    │
└─────────────────────────────────────────────────────────────────────┘
│
▼
┌──────────────────────────────────────────────┐
│  list_sources.html (PUBLIC)                  │
│  Displays: "List of Historical Sources"      │
│  Hero section populated by JS at page load   │
└──────────────────────────────────────────────┘

Stage 3 — Scan Site for Source Usage

Triggered manually or by agent ("run scrape" light action)
  │
  ▼
┌─────────────────────────────────────────────────────────────────────┐
│  source_auditor.js  ← JS script: source_auditor.js                 │
│                                                                     │
│  1. GET /api/v1/sources → master source list (all validated)       │
│  2. GET all content pages that cite sources inline:                 │
│     ├─ /records.html, /evidence.html, /context.html                │
│     ├─ /challenge.html, /resources.html                            │
│     ├─ Essay pages, Record pages, Response pages                   │
│     └─ Blog posts                                                  │
│  3. For each page, scan DOM for source references:                 │
│     ├─ Match by DOI string   (e.g. "10.1093/…")                   │
│     ├─ Match by ISBN string  (e.g. "978-0-300-…")                 │
│     ├─ Match by URL          (e.g. "jstor.org/…")                 │
│     └─ Match by title text   (fuzzy substring match)               │
│  4. Compare:                                                        │
│     ├─ ORPHANED: source in list_sources but cited on zero pages    │
│     ├─ MISSING:  citation on a page but not in list_sources        │
│     └─ HEALTHY:  source exists in both places                      │
│  5. Return audit report:                                            │
│     { orphaned: [], missing: [], healthy: [], total_pages_scanned } │
└─────────────────────────────────────────────────────────────────────┘

Valid Source Format

VALID SOURCE STRUCTURE:
═══════════════════════════════════════════════════════════════════════

  title ("by" author) (year) [identity]

WHERE:

  title = ONE OF:
    ├─ URL:      https://example.com/article         (clickable link)
    ├─ Book:     <i>The Historical Jesus</i>          (italic)
    └─ Article:  'Criteria of Authenticity'            (quoted)
                 + OPTIONAL: <i>Journal Name</i>      (italic journal)

  author =
    ├─ First Last:  John Meier                         (plain text)
    └─ ORCID:       0000-0002-1825-0097                (linked to orcid.org)

  year =
    └─ (2009)                                          (4-digit in parens)

  identity = ONE OF:
    ├─ DOI:   <span>doi:10.1093/oxfordhb/…</span>    (if article)
    ├─ ISBN:  <span>ISBN 978-0-300-…</span>           (if book)
    └─ URL:   <span>https://…</span>                  (if web source)

═══════════════════════════════════════════════════════════════════════

EXAMPLES:

  'Criteria of Authenticity' <i>Journal for the Study of the
  Historical Jesus</i> by John Meier (2009) <span>doi:10.1177/…</span>

  <i>Jesus: A New Vision</i> by Marcus Borg (1987)
  <span>ISBN 978-0-06-060911-2</span>

  https://www.deadseascrolls.org.il by <a href="orcid.org/0000-…">
  Emanuel Tov</a> (2014) <span>https://www.deadseascrolls.org.il</span>

File & Function Reference

Rust Types (app_core/src/types/system/source.rs)
TypeFields / VariantsDescription
Source id: Option<i64>, author: Author, title: SourceTitle Core source struct. WASM-safe via Serde derive. Stored in SQLite sources table.
Author Name(String), Orcid(String) Enum. If Orcid, the name becomes a link to orcid.org/{id}.
SourceTitle text: String, identity: Option<SourceIdentity> Title text + optional identity link appended as <span>.
SourceIdentity NamedUrl(String), Isbn(String), AcademicArticleId(String) Enum. Determines the <span> link type (DOI, ISBN, or URL).
SourceGatekeeper wraps Source Newtype enforcing validation: title ≤ 140 chars, non-empty author. Possession = validated.
SourceError EmptyTitle, TitleTooLong, InvalidAuthor, ValidationFailure(String) Error enum for gatekeeper rejections.
Rust Types — Potential New Types Needed
Proposed TypeLocationPurpose
SourceAuditReport app_core/src/types/system/source.rs Struct: orphaned: Vec<Source>, missing: Vec<String>, healthy: Vec<Source>, pages_scanned: u32. Return type for the audit endpoint.
PublicationYear app_core/src/types/system/source.rs Newtype wrapper: u16 with validation (1000–2100 range). Currently not stored on Source. Requires adding year: Option<PublicationYear> to Source struct + SQLite migration.
SourceType app_core/src/types/system/source.rs Enum: Book, Article, WebSource. Currently inferred from SourceIdentity variant but not stored explicitly. Adding this would simplify rendering logic.

All new types must: derive Serialize, Deserialize, Debug, Clone; be #[cfg_attr(target_arch = "wasm32", wasm_bindgen)] compatible; follow the 4-layer pattern (Skeleton → Brain → Gatekeeper → Errors).

API Endpoints
RouteMethodHandlerDescription
/api/v1/sources GET handle_get_sources() Returns all sources from SQLite. Used by list_sources_hero.js and audit script.
/api/v1/admin/sources POST handle_create_source() Creates a new source. Accepts CreateSourceRequest DTO, converts via TryFrom to core Source.
/api/v1/admin/sources/:id DELETE handle_delete_source() Deletes a source by SQLite row ID.
/api/v1/list_sources_content GET — Returns formatted sources for the public list_sources.html hero. Called by list_sources_hero.js.

Storage (SQLite)
MethodSQLDescription
get_sources() SELECT id, author_type, author_val, title_text, identity FROM sources Returns all sources. Reconstructs Author enum from author_type + author_val columns.
create_source() INSERT INTO sources (author_type, author_val, title_text, identity) … Persists a validated Source. Identity stored as JSON string.
delete_source() DELETE FROM sources WHERE id = ? Removes source by row ID.
