System Data Architecture

Reference for the /api/v1/system/feed endpoint and its 10 data categories.

Data Flow

System Feed Data Flow

[Browser] 
   -- (HTTP GET /api/v1/system/feed) --> 
[AppState] 
   -- (Read Access / RwLock) --> 
[SQLite Storage] 
   -- (10x Parallel SQL Queries) --> 
[Rust Structs] 
   -- (Serde / JSON Sterilization) --> 
[JSON Response]

The 10 Categories

+----------------------------------------------------------------------------+
| contacts       → SQLite: contacts (id: UUID)                               |
| drafts         → SQLite: essays, records, challenges (id: rowid)           |
| security       → SQLite: security_log (id: rowid)                          |
| work_queue     → In-memory: VecDeque<WorkItem> (capacity-gated)            |
| reflections    → SQLite: agent_reflections (id: rowid)                     |
| trace          → SQLite: trace_reasoning (trace_id: rowid)                 |
| spelling       → SQLite: spelling_issues (issue_id: rowid)                 |
| deadlinks      → SQLite: deadlinks (link_id: rowid)                        |
| page_metrics   → SQLite: page_metrics (page_path: TEXT)                    |
| server_metrics → SQLite: server_metrics (row_id: 1)                        |
+----------------------------------------------------------------------------+
