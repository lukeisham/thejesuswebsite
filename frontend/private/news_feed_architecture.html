Newsfeed Architecture

System architecture reference for the news crawler, storage pipeline, image handling, and public display. Read-only — no forms or API calls.

Data Flow

Diagram 1 — Monthly Crawl Pipeline

  TRIGGER (monthly schedule or manual dashboard button)
  wgt_news_crawler.js  ──POST /api/v1/news_run──►  api_news.rs :: handle_news_run()
                                                           │

                                  RSS     ├─YES──► scrape_domain(domain)      │

                                          │     │         │                  │ │

File & Function Reference

Config — Domain Sources
FileKeyDescription
news_sources.toml (project root)[[sources]]Array of source domains. Each entry has name, domain, rss_url, scrape_fallback
news_sources.tomlcrawl_interval_daysHow often the agent auto-crawls. Default: 30 (monthly)
Backend — Core Types (app/app_core/src/types/blog_and_news/news.rs)
Struct / FnKey Fields / SignatureDescription
NewsItemid, title, source_url, snippet, contents, picture_url: Option<Url>, harvested_atA processed, AI-summarised news article ready for display
RawNewsItemtitle, url, raw_content, raw_image_url: Option<String>Unprocessed article scraped from source; held in news_holding_area table pending AI summary
NewsSourcename, domain, rss_url, scrape_fallbackOne entry from news_sources.toml. Loaded at crawl time
NewsConfigcrawl_interval_days, sources: Vec<NewsSource>Top-level config struct. Load with NewsConfig::load(path)
NewsEngine::crawl_sources()(&self, config: &NewsConfig) -> Vec<RawNewsItem>Iterates sources; tries RSS, falls back to scraping
fetch_rss()(feed_url: &str) -> Result<Vec<RawNewsItem>>Fetches and parses RSS/Atom feed. Returns up to 10 items per source
scrape_domain()(domain_url: &str) -> Result<Vec<RawNewsItem>>Fetches domain HTML, extracts og:image, title, meta description
fetch_article_image()(article_url: &str, keywords: &str) -> Option<String>Tries og:image on article page; falls back to Unsplash API search
fetch_unsplash_image()(keywords: &str) -> Option<String>Queries Unsplash /search/photos with article keywords. Requires UNSPLASH_ACCESS_KEY env var
NewsGatekeeper::verify_url()(input: &str) -> Result<Url>Validates URL is HTTPS. Used to validate image URLs before storing
Backend — API Handlers (app/app_ui/src/api_news.rs)
HandlerRouteMethodDescription
handle_news_run()/api/v1/news_runPOSTTriggers the crawl pipeline. Crawls all sources, stores items, trims to 25
handle_get_news()/api/blog/newsGETReturns news items as JSON. Accepts optional ?limit=N query param
handle_news_feed_content()/api/v1/news_feed_contentGETReturns pre-rendered HTML fragment for the news feed hero section
Backend — Storage (app/app_storage/src/sqlite.rs)
MethodTableDescription
get_news_feed()news_itemsSELECT all items ORDER BY harvested_at DESC. Returns Vec<NewsItem>
store_news_item()news_itemsINSERT one processed article (7 fields including optional picture_url)
trim_news_feed_to_limit(limit)news_itemsDELETE rows not in the most recent N items. Enforces 25-item cap
get_harvested_news()news_holding_areaSELECT all raw unprocessed items
store_harvested_news()news_holding_areaINSERT raw article awaiting AI summary
delete_harvested_news_by_url()news_holding_areaDELETE processed item from holding area by URL
Frontend — JavaScript
FileFetchesRenders toDescription
frontend/js/display_news_feed.jsGET /api/blog/news#news-feed-listRenders full list of up to 25 items on news_feed.html. Each card: thumbnail + title + snippet + source link + date
frontend/js/display_top_four_news_items.jsGET /api/blog/news?limit=4#news-feedRenders 4 snippet cards on news_and_blog.html. Same card format as full feed
frontend/js/news_feed_hero.jsGET /api/v1/news_feed_content#hero-placeholderInjects pre-rendered HTML fragment into the hero area of news_feed.html
frontend/js/widgets/wgt_news_crawler.jsPOST /api/v1/news_runDashboard widget statusTriggers crawl manually or on schedule. Auto-poll interval: 30 days (CRAWL_INTERVAL_MS)
Frontend — Public Pages
PagePurposeNews Section

frontend/news_feed.htmlDedicated full news feed pageFull 25-item feed via display_news_feed.js and hero via news_feed_hero.js
Environment Variables
VariableRequiredDescription
