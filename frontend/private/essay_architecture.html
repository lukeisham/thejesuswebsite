Context Essays Architecture

System architecture reference for Context Essays. Read-only — no forms or API calls.

Step 1 — CRUD Creation (Markdown Editor)

 dashboard.html — Essays CRUD panel (id: crud-essays)
 ├─ #essay-title-input      "Link to Record title..." (text)
 ├─ #essay-record-id        hidden field — ULID of a linked Record (optional)
 ├─ #essay-body-input       8-row textarea — Markdown body
 │   Supported syntax:
 │   # H1  ## H2  ### H3   →  headers
 │   **Bold**  *Italic*    →  emphasis
 │   [Link text](url)      →  hyperlinks
 │   - Item                →  unordered list
 │   `inline code`         →  code
 │
 ├─ #save-essay-draft button
 │       │
 │       ▼
 │   essay_crud.js → getDraftPayload()
 │   POST /api/v1/essays/draft
 │   └─ sqlite.rs: save_essay_draft(draft)
 │       → stores in essay_drafts table (id, title, body, created_at)
 │
 └─ #publish-essay button
         │
         ▼
     essay_crud.js → getPublishPayload()
     POST /api/v1/essays/publish
     └─ api_essays.rs: handle_publish_essay()
         ├─ Essay::compose(title, author, text, …)
         │   └─ status: Unpublished (default)
         ├─ Essay::publish()
         │   └─ EssayGatekeeper::check_publication_readiness()
         │       ├─ title must not be empty or whitespace
         │       ├─ body must be ≥ 100 characters
         │       └─ if body contains [ → bibliography must not be empty
         ├─ status → Published
         │
         ├─ sqlite.rs: store_essay()      (structured storage)
         └─ chroma.rs: store_essay()      (vector embedding for search)

Step 2 — Search Tab (Browse Published Essays)

 dashboard.html — Essays panel
 ├─ Tab: "Write"    (crud-essays editor — step 1)
 └─ Tab: "Search"   (new tab — browse + manage essays)
         │
         ▼  (on tab open)
     essay_crud.js → loadEssays()
     GET /api/v1/essays   (Bearer auth header)
         │
         ▼
     renderEssayList(essays)
     ├─ Each row: title  |  status badge (Published / Draft)  |  ✕
     ├─ #essay-search-input
     │   └─ keyup → real-time filter by title text
     ├─ click row → populateForm(essay) → switch to Write tab
     │               (pre-fills all fields for editing)
     └─ ✕ button → deleteEssay(id)
                    DELETE /api/v1/essays/:id
                    └─ removes from sqlite.rs + chroma.rs

Step 3 — Internal URL (Slug Generation)

 AT PUBLISH TIME — slug generated from title
 ────────────────────────────────────────────
 essay.title = "The Life of Paul"
         │
         ▼
 slug = title
         .to_lowercase()          → "the life of paul"
         .replace(' ', '-')       → "the-life-of-paul"
         .strip non-alphanumeric  → "the-life-of-paul"
         │
         ▼
 internal_url = "/context/the-life-of-paul"
 stored alongside essay in sqlite.rs

 Examples:
 ├─ "Roman Occupation of Judea"  → /context/roman-occupation-of-judea
 ├─ "Who Were the Pharisees?"    → /context/who-were-the-pharisees
 └─ "Isaiah 53 & the Servant"   → /context/isaiah-53-the-servant

Step 4 — Public Display (context.html)

 context.html (page load)
 └─ context_hero.js (DOMContentLoaded)
         │
         ▼
    fetch('/api/v1/essays')   ← published essays only
         │
         ▼
    response HTML injected into:
    <section id="hero-placeholder" class="hero-container">

    Each essay card rendered as:
    ┌───────────────────────────────────┐
    │ [cover image if present]          │
    │ Title                             │
    │ by Author                         │
    │ excerpt (first ~150 chars)…       │
    │ [Read essay →] /context/<slug>   │
    └───────────────────────────────────┘

    Card grid: CSS grid, auto-fill, min 300px per column
    Error state: "No featured essay is currently active."

Glossary of Terms

TermDefinition
EssayA long-form Markdown article providing historical, cultural, or theological context for the evidence
CRUDCreate, Read, Update, Delete — the four operations available in the dashboard essay editor
MarkdownLightweight text formatting syntax used in the essay body textarea (headers, bold, italic, links, lists, code)
DraftAn unpublished essay saved to the essay_drafts SQLite table; not visible on context.html
PublicationStatusEnum: Published (visible on context.html) | Unpublished (draft only); essays default to Unpublished at creation
SlugURL-safe version of the essay title, auto-generated at publish time (e.g. "The Life of Paul" → "the-life-of-paul")
Internal URLThe public path to a published essay: /context/<slug>
EssayGatekeeperRust validator ensuring publish-readiness: title non-empty, body ≥ 100 chars, bibliography required if citations detected
ChromaDBVector database storing embedded essay text (384-dim BERT) in the "essays" collection for semantic search
EntryToggleMetadata tag categorising content type — essays use the Context variant
context.htmlThe public essays page where all published essays appear as cards in the hero-placeholder grid
hero-placeholderThe <section id="hero-placeholder"> in context.html that receives dynamically injected essay card HTML from context_hero.js
